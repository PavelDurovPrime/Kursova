# GradeTier — Розширений рейтинг студентських оцінок

## 1. Опис проєкту

**GradeTier** — веб-застосунок на Node.js для перегляду студентського журналу, аналітики успішності та візуалізації навчальних досягнень.

Проєкт демонструє поєднання бекенд-логіки на Node.js та інтерактивного фронтенду, реалізуючи ключові принципи розробки ПЗ.

### Основні можливості:
- **Аналітика потоку**: автоматичний розрахунок середнього балу та відвідуваності
- **ECTS Оцінювання**: автоматична конвертація балів у систему ECTS (A, B, C, D, E, F)
- **Кольорове маркування**: візуальна індикація успішності (зелений > 90, жовтий 60-75, червоний < 60)
- **Деталізація по предметах**: перегляд історії занять, балів та пропусків з heatmap-календарем відвідуваності
- **Фільтрація та сортування**: гнучкий пошук за ПІБ, групою, семестром або предметом з auto-submit та debounce
- **Tier Badges**: ранги студентів (Diamond, Platinum, Gold) на основі середнього балу
- **Fire Streak**: індикатори серії успішних оцінок (5+ оцінок 95+)
- **Рейтинг груп**: окрема сторінка порівняння груп за середнім балом
- **Експорт звітів**: завантаження у форматі TXT 
- **Графіки трендів**: візуалізація прогресу через Chart.js

## 2. Архітектура та Технології

### Стек технологій:
| Шар | Технології |
|-----|-----------|
| **Backend** | Node.js, Express.js 4.x |
| **Frontend** | HTML5, CSS3 (CSS Variables), Vanilla JavaScript (ES6+) |
| **Charts** | Chart.js 4.x |
| **Data** | JSON-файл (`data/grades.json`) |
| **Validation** | AJV (JSON Schema) |

### Архітектурні патерни:

#### Layered Architecture (Шарова архітектура)
```
┌─────────────────────────────────────────┐
│  UI Layer (HTML/CSS/JS)                 │
│  - index.html, rating.html               │
│  - app.js, rating.js, styles.css         │
├─────────────────────────────────────────┤
│  Web API Layer (Express)                  │
│  - server.js (REST endpoints)              │
│  - reportApi.js, dataService.js            │
├─────────────────────────────────────────┤
│  Services Layer (Business Logic)          │
│  - gradeService.js (розрахунки, ECTS)     │
│  - reportExporter.js (HTML/TXT export)     │
│  - logger.js (логування)                  │
├─────────────────────────────────────────┤
│  Repository Layer (Data Access)           │
│  - gradeRepository.js (читання JSON)       │
├─────────────────────────────────────────┤
│  Models Layer (Entities)                  │
│  - Student.js, Grade.js                   │
└─────────────────────────────────────────┘
```

## 4. API Endpoints

| Method | Endpoint | Опис |
|--------|----------|------|
| GET | `/api/meta` | Метадані (групи, предмети, періоди) |
| GET | `/api/report` | Звіт з фільтрацією та сортуванням |
| GET | `/api/group-rating` | Рейтинг груп |
| GET | `/api/student/:id/grades` | Оцінки конкретного студента |
| GET | `/api/export?format=html\|txt` | Експорт звіту |

## 5. Робота з системою

1. **Перегляд звіту**: Виберіть фільтри (семестр, група) — дані оновляться автоматично (auto-submit з debounce 300ms)
2. **Деталі студента**: Натисніть на рядок студента в таблиці. У модальному вікні можна переглянути:
   - Список предметів з оцінками
   - Heatmap календар відвідуваності
   - Діаграму успішності по предметах
3. **Рейтинг груп**: Перейдіть на сторінку "Рейтинг груп" для порівняння груп
